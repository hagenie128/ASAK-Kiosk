## 2. 작업 목적

- 키오스크 프로토타입 화면 설계 방향을 잡기 위해 참고 디자인을 조사한다.
- 팀 회의를 통해 고객용·관리자용 화면의 디자인 방향을 확정한다.
- 확정된 방향을 바탕으로 Figma 시안과 관리자 카테고리 섹션 컴포넌트 제작을 시작한다.

## 3. 직접 구현 영역

- **키오스크 디자인 레퍼런스 리서치:** 키오스크 UI 사례를 조사하고 화면 구성·스타일별 참고 시안을 리스트업했다.
- **디자인 시안 선택 회의:** 팀원에게 레퍼런스를 공유하고 실제 화면 설계에 적용할 시안 방향을 논의했다.
- **고객·관리자 화면 디자인 시안 제작:** 채택된 레퍼런스를 바탕으로 고객용·관리자용 키오스크 화면 설계 시안을 제작하고 팀에 배포했다.
- **화면 설계 회의록 정리:** 회의 중 논의 사항과 결정 사항을 문서로 기록하고 공유했다.
- **관리자 카테고리 섹션 컴포넌트:** Figma에서 관리자 키오스크 기본 카테고리 섹션 컴포넌트를 제작했다.

## 4. 구현 로직 / 적용한 방식

- 레퍼런스는 고객 주문 흐름과 관리자 운영 흐름을 나눠 비교했다.
- 디자인 시안 선택 시 사용자 흐름, 화면 가독성, 컴포넌트 재사용 가능성을 기준으로 삼았다.
- 고객 화면은 메뉴 탐색과 주문 흐름이 자연스럽게 이어지도록 구성했다.
- 관리자 화면은 주문 관리, 메뉴 관리, 상태 변경 같은 운영 업무가 잘 보이도록 정보 구조를 잡았다.
- 카테고리 섹션은 이후 선택 상태, 편집 상태, 등록 상태로 확장할 수 있도록 컴포넌트화 방향을 고려했다.

## 6. 발생 이슈

- 이슈 1:
  - 증상: 관리자 화면의 카테고리 섹션은 이후 상태 변화가 많아질 가능성이 있음
  - 원인: 메뉴 관리 화면에서 선택, 등록, 수정 상태가 계속 추가될 수 있음
  - 해결: 기본 카테고리 섹션을 먼저 컴포넌트로 만들고 이후 상태 확장을 고려해 구조를 잡음
