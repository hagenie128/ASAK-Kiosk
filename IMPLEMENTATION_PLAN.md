# ASAK Kiosk React Mock 구현 계획 (2026-07-14 ~ 2026-07-22)

> 목표: 7월 22일까지 키오스크 주문 흐름 전체를 실제 서버 없이 React + mock 데이터로 시연 가능하게 만든다. 관리자 운영 화면은 `ASAK_Admin` 저장소의 범위이며 이 프로젝트에 구현하지 않는다.

## 0. 범위와 완료 기준

| 구분 | 화면 | React mock 완료 범위 |
| --- | --- | --- |
| 주문 시작 | `SCR-001` | 매장/포장 선택, 주문 초안에 `orderType` 저장 |
| 메뉴·옵션 | `SCR-003`, `SCR-004` | 카테고리, 메뉴, 옵션, 제외 재료, 품절 표현 |
| 장바구니 | `SCR-005` | 수량·옵션 변경, 총액, 주문 확인 모달 |
| 결제·완료 | `SCR-007`, `SCR-008` | 결제 대기, 성공, 주문번호, 주문 초안 초기화 |
| 결제 예외 | `SCR-012` | 실패 overlay/toast, 장바구니·옵션 보존 |

`SCR-002`는 `SCR-001`에, `SCR-006`은 `SCR-005` 확인 모달에 포함한다. `SCR-012`는 별도 페이지가 아니다.

완료 기준: 각 화면의 success/loading/empty/error 상태, mock 성공·실패 흐름, 키보드 조작, `npm run lint`, `npm run build`를 모두 통과한다.
