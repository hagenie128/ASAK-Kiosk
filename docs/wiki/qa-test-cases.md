## TC 상세

### TC-001 신규 고객의 기본 주문 흐름 성공

- **전제조건**: 키오스크 정상 구동, 모든 메뉴 판매중
- **수행 절차**: 1) 홈 진입 2) 메뉴 선택 3) 추천조합+토핑 4) 장바구니 5) 결제
- **기대 결과**: 결제 완료 후 주문번호 표시
- **관련**: SC-001 · API-001~006 · SCR-001~008

### TC-002 재방문 고객 빠른 주문(화면 5회 이내)

- **전제조건**: 재방문 고객, 추천조합 기본값 설정됨
- **수행 절차**: 메뉴선택→옵션(기본값 유지)→장바구니→주문확인→결제, 화면 전환 5회 이내
- **기대 결과**: 주문번호 표시, 화면 전환 ≤5회
- **관련**: SC-002 · API-002~006 · SCR-003~008

### TC-003 품절 항목 포함 시 주문 불가/대체

- **전제조건**: 관리자가 재료/옵션 품절 처리
- **수행 절차**: 1) API-009 품절 ON 2) 키오스크 메뉴/옵션 확인 3) SOLD OUT 표시 4) 대체 선택
- **기대 결과**: 품절 항목 선택 불가, 다른 항목으로 주문 가능
- **관련**: SC-003 · API-002~004, API-009 · SCR-003~004, SCR-011

### TC-004 결제 실패 시 원인 안내·장바구니 유지

- **전제조건**: 결제 실패 테스트 모드
- **수행 절차**: 1) 장바구니 담기 2) 결제 3) 실패 응답 4) 안내 확인 5) 장바구니 유지
- **기대 결과**: 실패 원인 표시, 장바구니 유지
- **관련**: SC-005 · API-006 · SCR-007, SCR-012

### TC-005 결제 성공 및 주문번호 생성

- **전제조건**: 장바구니 1건 이상
- **수행 절차**: 주문확인→CARD 결제→승인
- **기대 결과**: payment APPROVED, orderNo 생성
- **관련**: SC-004 · API-005~006 · SCR-006~008

### TC-006 멤버십 스탬프 1회 확인 후 적립

- **전제조건**: 멤버십 보유 고객 (확장)
- **수행 절차**: 결제 중 1회 확인→결제 완료→적립 표시
- **기대 결과**: 이중 확인 없이 적립 결과 표시
- **관련**: SC-006 · API-018 · SCR-007~008

### TC-007 관리자 주문 상태 변경

- **전제조건**: 결제 완료 주문 존재
- **수행 절차**: 목록→상세→RECEIVED→PREPARING→COMPLETED
- **기대 결과**: DB·화면 상태 일치
- **관련**: SC-008 · API-007~008 · SCR-009~010

### TC-008 접근성 옵션 적용 주문

- **전제조건**: 저시력 고객 시나리오 (후반)
- **수행 절차**: 글자크기 확대→메뉴 탐색→주문 완료
- **기대 결과**: 안내 없이 주문 완료
- **관련**: SC-013 · API-017 · SCR-014

### TC-009 관리자 로그인 검증

- **전제조건**: 관리자 계정 (후반)
- **수행 절차**: 로그인→세션 발급→관리자 화면 진입
- **기대 결과**: 인증 성공/실패 분기 (Notion row 미등록)
- **관련**: — · API-HOLD-001 · SCR-015

### TC-010 관리자 메뉴 목록 조회

- **전제조건**: 관리자 메뉴 관리 화면
- **수행 절차**: 목록 조회→필터/품절 표시 확인
- **기대 결과**: seed 메뉴 84건 조회
- **관련**: SC-017 · API-011 · SCR-016

### TC-011 관리자 신규 메뉴 등록

- **전제조건**: 관리자 로그인 (후반)
- **수행 절차**: 메뉴 등록→옵션그룹 연결→저장→키오스크 노출
- **기대 결과**: 키오스크 메뉴 목록 즉시 반영
- **관련**: SC-017 · API-012 · SCR-017

### TC-012 관리자 판매 항목 품절 관리

- **전제조건**: 관리자 품절 화면
- **수행 절차**: 대상 선택→품절 토글→키오스크 반영 확인
- **기대 결과**: CORE/BASE/DEFAULT/OPTION 품절 규칙 준수
- **관련**: SC-007 · API-009~010 · SCR-011

### TC-013 관리자 일별 매출 조회

- **전제조건**: 주문·결제 데이터 존재 (후반)
- **수행 절차**: 기간 선택→일별 매출 조회
- **기대 결과**: 일자별 주문수·금액 표시
- **관련**: SC-018 · API-015 · SCR-019

### TC-014 최종 통합 리허설(E2E)

- **전제조건**: 데모 환경 준비
- **수행 절차**: 고객 주문→결제→관리자 확인/상태변경 일괄 시연
- **기대 결과**: 흐름 끊김 없이 시연 가능
- **관련**: SC-024 · API-001~009 · SCR-001~011
