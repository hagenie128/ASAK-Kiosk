## 2. 작업 목적

- 이번 작업에서 해결하려고 한 문제: 메뉴 옵션 선택에 따라 가격이 바로 바뀌지 않아 사용자 입장에서 총액 확인이 어려웠다.
- 기대 결과: 옵션과 수량을 바꾸면 화면에 표시되는 가격과 장바구니 반영 가격이 즉시 일치해야 한다.

## 3. 직접 구현 영역

- 옵션 선택 상태를 관리하는 프론트 구조를 직접 정리했다.
- 기본 메뉴 가격 + 추가 옵션 가격을 합산하는 계산 로직을 직접 구현했다.
- 필수 옵션이 선택되지 않으면 장바구니 버튼을 비활성화하는 조건을 직접 추가했다.
- 장바구니에 담길 때 선택 옵션 목록과 최종 가격이 함께 저장되도록 데이터 구조를 맞췄다.

## 4. 구현 로직 / 적용한 방식

- 어떤 로직을 사용했는지:
  - 선택된 옵션 목록을 배열로 저장하고, 옵션/수량이 바뀔 때마다 합계를 다시 계산하는 방식 사용
- 왜 그 방식을 선택했는지:
  - 현재 화면 규모에서는 복잡한 전역 상태관리보다 컴포넌트 내부 상태와 계산 함수 조합이 더 단순하고 유지보수하기 쉬웠다.
- 데이터 흐름 또는 처리 순서:
  - 메뉴 선택
  - 옵션 선택/해제
  - 상태 업데이트
  - 총액 재계산
  - 장바구니 데이터 생성
- 핵심 조건문 / 상태 변경 / 예외 처리 방식:
  - 필수 옵션이 없으면 주문 버튼 비활성화
  - 수량은 1 미만으로 내려가지 않게 방지
  - 옵션 데이터가 비어 있거나 null이어도 기본 가격만으로 계산 가능하게 처리

## 5. AI 도움 영역

- 사용한 AI 도구: Cursor
- 어떤 질문/요청을 했는지:
  - "React에서 옵션 선택 상태와 합계 계산 구조를 단순하게 짜는 방법 제안해줘"
- AI가 도움 준 내용:
  - 상태 구조 초안
  - 합계 계산 함수 분리 아이디어
  - 필수 옵션 검증 포인트 제안
- 그대로 사용한 부분:
  - 계산 함수 분리 방식 아이디어
- 수정해서 사용한 부분:
  - 상태 필드 구조
  - 장바구니에 저장하는 데이터 형태
  - 버튼 비활성화 조건

## 6. 발생 이슈

- 이슈 1:
  - 증상: 옵션을 바꿔도 화면 가격은 바뀌는데 장바구니에는 이전 가격이 들어갔다.
  - 원인: 화면 표시용 계산값과 장바구니 저장용 계산값이 다른 시점에 계산되고 있었다.
  - 해결: 장바구니 저장 직전에 동일한 계산 함수를 재사용하도록 구조를 통일했다.

- 이슈 2:
  - 증상: 옵션 배열이 비어 있을 때 콘솔 에러가 발생했다.
  - 원인: `map` 호출 전 null 체크가 없었다.
  - 해결: 기본 빈 배열 처리와 조건 분기를 추가했다.

## 7. 디버깅 기록

- 확인한 로그/에러 메시지:
  - `Cannot read properties of null (reading 'map')`
- 의심했던 지점:
  - 옵션 데이터 파싱
  - 상태 초기화
  - 장바구니 생성 함수
- 실제 원인:
  - 옵션 데이터가 없는 메뉴에서도 동일한 렌더링 로직을 타고 있었고, null 체크가 빠져 있었다.
- 다시 같은 문제가 생기면 먼저 볼 파일/명령어:
  - `MenuOptionPanel`
  - `CartItemBuilder`
  - 브라우저 Network 탭 응답 데이터
  - 브라우저 Console 에러 로그

## 8. 이번 작업에서 배운 점

- 새로 배운 기술/개념:
  - React에서 화면 표시용 값과 저장용 값을 다른 로직으로 두면 상태 불일치가 생기기 쉽다는 점
- 이번에 이해가 선명해진 부분:
  - 상태 변경 직후 값을 바로 믿기보다, 공통 계산 함수를 두고 재사용하는 편이 안전하다는 점
- 다음 작업에 적용해보고 싶은 점:
  - 장바구니, 결제 예상 금액, 할인 계산에도 같은 방식의 공통 계산 함수 적용

## 10. 검증 내용

- 실행한 명령어:
  - `npm run dev`
- 테스트한 시나리오:
  - 메뉴 선택 후 옵션 변경
  - 수량 증가/감소
  - 필수 옵션 미선택 상태에서 버튼 확인
  - 장바구니에 담은 뒤 총액 비교
- 확인 결과:
  - 화면 가격과 장바구니 가격이 일치함
  - 필수 옵션 미선택 시 버튼 비활성화 정상
  - 옵션 없는 메뉴도 에러 없이 동작

## 11. 포트폴리오용 요약
