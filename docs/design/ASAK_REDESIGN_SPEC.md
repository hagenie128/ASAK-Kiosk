## 5. 4열 메뉴 규칙

- [ ] 기준 폭 `960`에서 4열을 고정한다.
- [ ] 좌우 마진 `60`을 유지한다.
- [ ] 열 사이 거터는 `24`로 유지한다.
- [ ] 카드는 항상 동일 폭 `222`를 사용한다.
- [ ] 세로 스크롤 상황에서도 카드 폭은 유지하고 행만 추가한다.
- [ ] 메뉴명이 2줄을 초과하면 말줄임 처리한다.
- [ ] 가격과 CTA는 카드 하단에 고정한다.
- [ ] 카테고리 탭은 그리드 상단에 배치하고 그리드와 `24` 간격을 둔다.

## 10. 만드는 방법

### 1단계. Figma 프레임 만들기

1. 키오스크 화면은 `1080 x 1920` 프레임으로 생성한다.
2. 관리자 화면은 `1920 x 1080` 프레임으로 생성한다.
3. 키오스크 프레임에는 좌우 `60` 마진 가이드를 둔다.
4. 관리자 프레임에는 좌측 `240` 사이드바와 상단 `72` 툴바 가이드를 둔다.

### 2단계. 키오스크 기본 구조 만들기

1. 상단 헤더 영역 `140`을 만든다.
2. 본문 콘텐츠 영역은 폭 `960` 안에서 구성한다.
3. 카테고리 탭은 메뉴 그리드 위에 배치한다.
4. 메뉴 카드는 `222 x 292` 컴포넌트로 만든다.
5. 메뉴 그리드는 4열, 거터 `24`로 반복 배치한다.
6. 하단 CTA 영역은 `180` 높이 안에서 고정한다.

### 3단계. 키오스크 메뉴 카드 만들기

1. 카드 프레임을 `222 x 292`로 만든다.
2. 상단 썸네일 영역을 `222 x 168`로 둔다.
3. 내부 패딩은 `16`으로 설정한다.
4. 메뉴명, 설명, 가격을 세로로 배치한다.
5. 메뉴명은 최대 2줄까지만 노출한다.
6. 가격과 추가 버튼은 카드 하단에 고정한다.

### 4단계. 관리자 기본 구조 만들기

1. 좌측 사이드바를 `240` 폭으로 만든다.
2. 로고 영역은 `80`, 1차 메뉴는 높이 `44`로 만든다.
3. 상단 툴바는 높이 `72`로 만든다.
4. 콘텐츠 영역은 좌우 패딩 `32`를 적용한다.
5. 콘텐츠 내부는 12열 그리드, 거터 `16`을 적용한다.
6. 화면 성격에 따라 보드, 목록+상세, KPI+테이블 중 하나를 기본 패턴으로 잡는다.

### 5단계. 관리자 화면별 제작 패턴

| 화면 | 제작 패턴 |
| --- | --- |
| 주문 현황 | 상태별 컬럼 보드 + 주문 카드 |
| 주문 관리 | 주문 목록 테이블 + 우측 상세 패널 |
| 품절 관리 | 상품 목록 + 판매상태/품절사유 편집 패널 |
| 메뉴 관리 | 상품 목록 + 옵션그룹 편집 패널 |
| 매출 요약 | KPI 카드 + 차트/테이블 |
| 결제 설정 | 결제수단 카드 + 정책 안내/수정 폼 |

### 6단계. 개발 구현 기준

1. 키오스크는 `max-width: 960px`, `margin-inline: 60px`에 해당하는 레이아웃 기준을 유지한다.
2. 메뉴 그리드는 CSS 기준 `grid-template-columns: repeat(4, 222px)`와 `gap: 24px`로 시작한다.
3. 메뉴 카드의 제목은 2줄 말줄임을 적용한다.
4. 관리자 콘텐츠는 사이드바 `240px`, 툴바 `72px`, 콘텐츠 패딩 `32px`를 상수화한다.
5. 관리자 테이블 행 높이 `56px`, 헤더 `48px`, 버튼 `44px`을 공통 컴포넌트 기준으로 둔다.
6. 주문/메뉴/결제 상태값은 더미 UI 텍스트가 아니라 실제 데이터 필드와 연결한다.

### 7단계. 최종 검수 순서

1. 화면 크기와 마진이 맞는지 확인한다.
2. 4열 메뉴 카드 폭과 거터를 확인한다.
3. 터치 대상의 최소 높이가 `88` 이상인지 확인한다.
4. 관리자 12열 그리드와 사이드바/툴바 크기를 확인한다.
5. 주문, 메뉴, 품절, 결제 설정에 필요한 실제 데이터 필드가 빠지지 않았는지 확인한다.
6. 결제 실패, 타임아웃, 영수증 선택 같은 예외 장면이 포함되어 있는지 확인한다.

## 16. 컴포넌트 제작 방법

### 1단계. 공통 토큰 먼저 만들기

1. `radius`, `spacing`, `text`, `height` 토큰을 먼저 정리한다.
2. 키오스크와 관리자가 같이 쓰는 값은 공통 토큰으로 둔다.
3. 화면별 특수 값은 컴포넌트 토큰으로 분리한다.

### 2단계. 키오스크 핵심 컴포넌트 만들기

1. Top Bar, CTA, Menu Card를 먼저 컴포넌트화한다.
2. Menu Card는 실제 권장 크기 `222 x 292` 기준으로 만든다.
3. Option Card와 Cart Item Card는 선택/수량 변경 상태를 포함한다.
4. Payment Method Card는 카드, 간편결제, 쿠폰을 같은 컴포넌트 변형으로 관리한다.

### 3단계. 관리자 핵심 컴포넌트 만들기

1. Sidebar, Toolbar, Filter Bar를 먼저 만든다.
2. Order Card, KPI Card, Detail Panel을 상태별로 변형한다.
3. Data Table은 Header와 Row를 분리해서 만든다.
4. Empty, Error, 결제 실패 같은 예외 상태를 별도 컴포넌트로 둔다.

### 4단계. 아이콘과 텍스트 스타일 연결하기

1. 아이콘 크기는 키오스크 24 중심, 관리자 20 중심으로 맞춘다.
2. 모든 버튼/카드/테이블에는 위 텍스트 스타일을 적용한다.
3. 라인하이트는 권장 라인하이트 표 기준으로 맞춘다.
4. 폰트는 `Noto Sans KR` Bold, Medium, Regular만 우선 사용한다.
